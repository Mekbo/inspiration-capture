# 灵感记录模板

此模板用于生成结构化的灵感记录。每条记录保存为一个独立的 Markdown 文件,使用 YAML frontmatter 存储元数据。

## YAML Frontmatter 字段说明

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `title` | string | 灵感标题（≤30字） | - |
| `type` | string | 灵感分类 | 产品创意 / 内容创意 / 技术方案 / 文章灵感 / 商业模式 / 设计灵感 / 个人成长 / 问题与痛点 / 观察与洞见 / 其他 |
| `tags` | list | 标签（3-8个） | - |
| `priority` | string | 优先级 | 高 / 中 / 低 |
| `created` | string | 创建时间 | ISO 8601 格式 |
| `status` | string | 状态 | 灵感 / 待验证 / 进行中 / 已完成 / 已放弃 |
| `source` | string | 灵感来源 | 阅读 / 对话 / 观察 / 梦境 / 散步 / 社交媒体 / 工作 / 其他 |

## 正文模板

```markdown
## 灵感来源
[是什么触发这个想法？一句话描述]

## 核心内容
[详细展开：问题、解决方案、独特价值、关键洞察]

## 应用场景
[这个灵感可以用在哪里？谁会是受众？]

## 待验证假设
- [假设1：需要验证的关键前提]
- [假设2]

## 下一步行动
- [ ] [具体的可执行步骤1]
- [ ] [具体的可执行步骤2]

## 关联参考
- [相关链接、文章、书籍、人物等]
```

## 文件命名规范

```
{workspace}/灵感库/{YYYY-MM-DD}-{简短标题}.md
```

- 工作区路径：从当前会话的 `Workspace Folder` 动态获取，拼接 `灵感库` 子目录
- 不要使用硬编码的绝对路径，始终基于当前工作区根路径构造
- 日期使用创建当天的日期
- 短标题从 title 字段提取，移除特殊字符，用连字符替代空格

## 分类判定规则

根据以下特征自动判定灵感类型：

| 类型 | 关键词/特征 |
|------|------------|
| 产品创意 | 功能、工具、APP、小程序、平台、服务、解决方案 |
| 内容创意 | 文章、视频、播客、小红书、公众号、短视频、选题 |
| 技术方案 | 架构、代码、框架、算法、API、自动化、系统设计 |
| 文章灵感 | 观点、论证、案例、故事、写作、论述 |
| 商业模式 | 盈利、变现、收入、增长、获客、转化 |
| 设计灵感 | UI、UX、配色、排版、视觉、交互 |
| 个人成长 | 习惯、学习、效率、健康、反思、目标 |
| 问题与痛点 | 痛点、不便、不爽、需求、槽点 |
| 观察与洞见 | 趋势、现象、规律、发现、洞察 |
| 其他 | 不匹配以上任何类型时默认归类 |
